# HRM（人力资源管理）

## 双含义

1. **Human Resource Management**（人力资源管理）：企业中"选、育、用、留"的完整管理活动——招聘、培训、绩效、薪酬、劳动关系
2. **Human Resource Management System**（人力资源管理系统）：将上述活动信息化的软件

## 核心模块

| 模块 | 做什么 |
|------|--------|
| **组织人事** | 组织架构、岗位编制、员工档案、入转调离全生命周期 |
| **薪酬福利** | 薪资计算、社保公积金、个税、工资条 |
| **考勤假期** | 打卡、加班、调休、年假额度管理 |
| **招聘** | 简历管理、面试流程、Offer 审批 |
| **绩效** | KPI/OKR 设置、考核评分、绩效面谈记录 |
| **培训** | 课程管理、学习记录、证书管理 |
| **干部管理** | 任命、轮岗、继任计划（大型企业特有） |

## 国内常见产品

北森、用友 DHR、薪人薪事、钉钉/飞书内置 HR 模块。

## 与 OA 考勤模块的边界

- **OA 管审批流**：请假申请提交、领导审批、流程流转
- **HRM 管余额和核算**：还剩几天年假、请假扣多少钱、薪资计算

两者通过接口同步——OA 审批通过后自动同步到 HRM 扣减余额。

## 与 ERP/CRM 的关系

详见 [[erp-crm-hrm-relationship|ERP vs CRM vs HRM 三者关系]]。核心定位：ERP 管企业内部资源（钱/物/产能），CRM 管外部关系（客户/商机），HRM 管人力资源（人/能力）。三者通过销售订单→生产→薪酬分摊的链路联动。

**Why:** 与 [[oa-system-overview|OA]]、ERP、CRM 构成企业数字化的四根支柱。FDE 在实施任一系统时都必然涉及它们之间的集成。

**How to apply:** 面试/工作中被问到企业系统全景时，用"ERP 管钱物、CRM 管客户、HRM 管人、OA 管流程"四句话框架定位；实施 HRM 时注意与 OA 考勤的边界划分。
