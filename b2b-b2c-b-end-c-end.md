## B 端 vs C 端

| | B 端（To Business） | C 端（To Consumer） |
|---|---|---|
| 客户 | 企业、组织、政府 | 个人用户 |
| 决策者 | 使用者 ≠ 付费决策者（老板/采购） | 使用者 = 付费者 |
| 决策逻辑 | 理性：ROI、效率、合规 | 感性+理性：体验、价格、品牌 |
| 销售周期 | 数周到数月，需 demo/招标/谈判 | 几分钟到几天 |
| 客单价 | 高（几万到千万） | 低（几块到几百） |
| 产品复杂度 | 高，需培训/定制/部署 | 低，开箱即用 |
| 界面要求 | 功能优先 | 体验优先，必须好看好用 |
| 典型案例 | 企业微信/飞书/Salesforce/用友/OA/ERP/HRM | 微信/抖音/淘宝/网易云音乐 |

## 常见变种
- **B2B**：企业卖企业
- **B2C**：企业卖个人
- **C2C**：个人卖个人（淘宝/闲鱼）
- **B2B2C**：企业通过另一企业触达消费者（连锁店管理系统）
- **G 端**：To Government——B 端超集，周期更长、合规更严、招投标为主
- **SMB**：中小企业——决策快、客单价中等、对开箱即用要求高

## 区分意义
B 端和 C 端能力模型完全不同：
- C 端思维做 B 端：觉得"产品好用户自然来"——B 端需要销售/BD/渠道
- B 端思维做 C 端：功能堆满、入口太深——C 端用户三秒不会用就走了
- FDE（前线部署工程师）只存在于 B 端——C 端不需要人到现场
